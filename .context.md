# TER OS Context & Architecture Overview

**Status:** Stable (v1.2)
**Purpose:** A modular, portable Termux shell environment and system integration toolkit.

## 1. Project Architecture
The project is split into two parts:
1. **Source Repository (`~/ter/`)**: Where all development happens. Contains the raw scripts.
2. **Active Environment (`~/.shell.d/`)**: The live installed environment on the user's Android device. Changes made in `~/ter/` must be deployed to `~/.shell.d/` (or run `bash ~/ter/install.sh`) to take effect.

## 2. Core Modules
- **`core/`**: Styling (`00-style.sh`), config, prompt themes, alias manager (`am`), and the dynamic App Registry loader (`app-loader.sh`).
- **`user/`**: Primary shortcuts, startup hooks (`welcome.sh`), tab titling, and ADB utilities (`adb-sysinfo`, `adb-manage`, `adb-audit`, etc.).
- **`network/`**: `adb_connect.sh` (`adbcon`) and subnet scanners (`scan.sh`).
- **`docs/`**: Comprehensive `cli_manual.md` containing usage for all tools.
- **`Tmux/`**: Mobile-optimized tmux config (`~/.tmux.conf`) using `~` as prefix, with touch-friendly ⏭ tab buttons and customized kinetic scroll overrides.

## 3. Background Stability Engine (`optimize`)
- **Location:** `~/ter/user/optimize.py` (wrapped by `optimize.sh`)
- **Function:** Solves Android 12+ Phantom Process Killer issues and battery hibernation.
- **Commands:** `optimize status`, `optimize fix`, `optimize run`, `optimize list`, `optimize stop`.
- **Logic:** Requires an active local ADB loopback. Uses `device_config` and `settings put global` to raise process limits to 2048. Checks gracefully handle offline ADB (`ADB OFFLINE`) or unset values (`EMPTY (DEFAULT 32)`).

## 4. ADB Connection Wizard (`adbcon`)
- **Location:** `~/ter/network/adb_connect.sh`
- **Function:** Automates Wireless Debugging pairing and port connection, then locks the server to the offline `127.0.0.1:5555` loopback.
- **Logic:** 
  - Explicitly filters for true Wi-Fi/Hotspot (`wlan`, `swlan`, `ap`), strictly ignoring mobile data (`rmnet`).
  - Checks `init.svc.adbd` to see if Developer Options / Debugging is globally enabled.
  - Instead of blocking the user aggressively, it provides clear, non-blocking step-by-step instructions before offering Pair/Connect/Auto-scan.

## 5. App Registration System
Third-party integrations (like NEXUS) can inject themselves into TER without modifying TER's core.
- Drops files into `~/.shell.d/apps/<app-name>/`.
- Must include a `manifest.json`.
- Handled by `apps list`.

## 6. Development Workflow (Context for AI)
- When modifying code, edit the files in `~/ter/`.
- After modifying, always deploy the changes to the active environment via `cp -r ~/ter/* ~/.shell.d/` or `bash ~/ter/install.sh`.
- Instruct the user to run `re` (Reload) in their terminal to apply shell changes.
- Always run `python3 ~/ter/make_motd.py` when changing the MOTD dashboard, which writes to `/usr/etc/motd`.
